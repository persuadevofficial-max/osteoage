# Software Requirements Specification (SRS)
# OsteoAge - 골연령 분석 시스템

**문서 버전:** 1.0
**작성일:** 2025-12-17
**준거 규격:** IEC 62304 (Medical Device Software - Software Life Cycle Processes)

---

## 1. 서론

### 1.1 목적
본 문서는 OsteoAge 골연령 분석 시스템의 소프트웨어 요구사항을 정의합니다.

### 1.2 범위
- 의료진용 웹 애플리케이션
- 관리자용 대시보드
- X-ray 이미지 기반 골연령 분석
- 성장 예측 및 리포트 생성

### 1.3 의료기기 분류
- Class: IIa (중위험 의료기기)
- 용도: 진단 보조 소프트웨어

---

## 2. 시스템 개요

### 2.1 시스템 설명
OsteoAge는 소아의 손 X-ray 이미지를 기반으로 골연령을 분석하고 성인 예측 키를 계산하는 웹 기반 의료 소프트웨어입니다.

### 2.2 사용자
- 주 사용자: 소아과 전문의, 성장클리닉 의사
- 부 사용자: 병원 관리자

---

## 3. 기능 요구사항 (Functional Requirements)

### FR-001: 사용자 인증
**설명:** 시스템은 안전한 사용자 인증을 제공해야 한다.
**우선순위:** 높음
**관련 위험:** RSK-001, RSK-002

**상세 요구사항:**
- FR-001-1: 이메일/비밀번호 기반 로그인
- FR-001-2: 역할 기반 접근 제어 (의사/관리자)
- FR-001-3: 세션 관리 및 자동 로그아웃

### FR-002: 환자 정보 관리
**설명:** 의사는 환자 기본 정보를 입력하고 조회할 수 있어야 한다.
**우선순위:** 높음
**관련 위험:** RSK-003

**상세 요구사항:**
- FR-002-1: 환자명, 성별, 생년월일, 신장, 체중 입력
- FR-002-2: 환자 ID 자동 생성
- FR-002-3: 과거 기록 조회

### FR-003: 골연령 분석
**설명:** 시스템은 의사가 입력한 골연령 정보를 기반으로 분석을 수행해야 한다.
**우선순위:** 높음
**관련 위험:** RSK-004, RSK-005

**상세 요구사항:**
- FR-003-1: 골연령 연/월 단위 입력 (1-18세 범위)
- FR-003-2: 생물학적 나이와 골연령 차이 계산
- FR-003-3: X-ray 이미지 표시 및 저장

### FR-004: 성장 예측
**설명:** 시스템은 다양한 요소를 고려하여 성인 예측 키를 계산해야 한다.
**우선순위:** 높음
**관련 위험:** RSK-005

**상세 요구사항:**
- FR-004-1: 현재 키 입력 (100-220cm 범위)
- FR-004-2: 유전적 예측 키 입력 (부모 키 기반)
- FR-004-3: 성장곡선 기반 예측 키 입력
- FR-004-4: 통합 예측 알고리즘 적용
  - 공식: 기본예측 + 유전보정(15%) + 성장곡선보정(10%)
- FR-004-5: 예측 결과의 백분위 표시

### FR-005: 성장 차트 시각화
**설명:** 시스템은 환자의 성장 데이터를 시각적으로 표현해야 한다.
**우선순위:** 중간
**관련 위험:** RSK-006

**상세 요구사항:**
- FR-005-1: 표준 성장곡선 (3%, 50%, 97% 백분위) 표시
- FR-005-2: 과거 성장 기록 점 표시
- FR-005-3: 현재 위치 및 골연령 기준 위치 표시
- FR-005-4: 예측 성인키 위치 표시
- FR-005-5: 성장 궤적 선 표시

### FR-006: 분석 리포트 생성
**설명:** 시스템은 분석 결과를 A4 형식 리포트로 출력해야 한다.
**우선순위:** 높음
**관련 위험:** RSK-007

**상세 요구사항:**
- FR-006-1: 환자 기본 정보 포함
- FR-006-2: 골연령 분석 결과 포함
- FR-006-3: 성장 차트 포함
- FR-006-4: PDF 출력 및 인쇄 기능

### FR-007: 관리자 기능
**설명:** 관리자는 시스템 전체를 모니터링하고 관리할 수 있어야 한다.
**우선순위:** 중간
**관련 위험:** RSK-002

**상세 요구사항:**
- FR-007-1: 회원 관리 (조회, 승인, 차단)
- FR-007-2: 사용 현황 통계 조회
- FR-007-3: 분석 로그 조회
- FR-007-4: 시스템 설정 관리

---

## 4. 비기능 요구사항 (Non-Functional Requirements)

### NFR-001: 성능
- NFR-001-1: 페이지 로딩 시간 3초 이내
- NFR-001-2: 예측 계산 응답 시간 1초 이내
- NFR-001-3: 동시 사용자 100명 지원

### NFR-002: 보안
- NFR-002-1: 환자 데이터 암호화 저장
- NFR-002-2: HTTPS 통신 필수
- NFR-002-3: 세션 타임아웃 30분
- NFR-002-4: 비밀번호 정책 준수 (최소 4자 - 개발 단계)

### NFR-003: 사용성
- NFR-003-1: 직관적인 UI/UX
- NFR-003-2: 다크 테마 적용
- NFR-003-3: 반응형 웹 디자인 (PC Web 최적화)

### NFR-004: 호환성
- NFR-004-1: Chrome, Safari, Edge 최신 버전 지원
- NFR-004-2: 최소 해상도 1280x720

### NFR-005: 신뢰성
- NFR-005-1: 시스템 가용성 99.5% 이상
- NFR-005-2: 데이터 백업 일 1회
- NFR-005-3: 에러 로깅 및 모니터링

---

## 5. 데이터 요구사항

### DR-001: 환자 데이터
- 환자 ID, 이름, 성별, 생년월일
- 신장, 체중, 측정일
- X-ray 이미지 경로

### DR-002: 분석 데이터
- 골연령 (년, 월)
- 현재 키, 유전 예측 키, 성장곡선 예측 키
- 최종 예측 키
- 분석 설명 텍스트

### DR-003: 사용자 데이터
- 사용자 ID, 이메일, 비밀번호 해시
- 역할 (doctor/admin)
- 병원명, 전화번호

---

## 6. 인터페이스 요구사항

### IR-001: 사용자 인터페이스
- Web-based GUI
- 반응형 디자인
- 다크 테마

### IR-002: 데이터 인터페이스
- LocalStorage/SessionStorage (개발 단계)
- REST API (향후 백엔드 연동)
